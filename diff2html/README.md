# diff2html 要件整理

## 目的
このスクリプトの目的は、Markdown ファイルの差分（Unified Diff）を入力として、HTML で差分を視覚化することです。

## 入力と出力
- 入力: Markdown の差分ファイル（例: `sample.diff`）
- 出力: 差分を可視化した HTML レポート（例: `rendered-diff-report.html`）

## 差分ファイルの前提条件
- 1つの差分ファイルに、Markdown 以外のファイル差分が含まれる場合がある。
- 1つの差分ファイルに、複数の Markdown ファイル差分が含まれる場合がある。
- 本スクリプトでは、Markdown（`.md`）以外のファイル差分は無視する。
- 本スクリプトでは、`.md` ファイルごとに差分を解析して可視化する。
- 本スクリプトでは、以下のケースにも対応する。
	- 新規作成された `.md` ファイル（追加ファイル）
	- 削除された `.md` ファイル（削除ファイル）

## 表示方針

### 1. 通常テキストブロック
- 一般的な差分ビューとして表示する。
- 差分箇所は、取り消し線や色のハイライトを使って視覚化する。
- 追加・削除・変更がひと目で分かることを重視する。

### 2. テーブルブロック
- テーブル内部のセル単位・文字単位で差分を強調しない。
- テーブル内に差分が 1 箇所でもあれば、テーブル全体をブロックとしてハイライトする。
- テーブルは HTML の `<table>` としてレンダリングする。

### 3. コードブロック
- コードブロック内部の行単位・文字単位で差分を強調しない。
- コードブロック内に差分が 1 箇所でもあれば、コードブロック全体をハイライトする。
- コードブロックは HTML の `<pre><code>` としてレンダリングする。

## 重要なルール
- 通常テキストは「行ベース差分」を維持する。
- テーブルとコードブロックは「ブロックベース強調」を適用する。
- 「どこが違うか」を細かく見せる対象は通常テキストのみ。
- テーブルとコードブロックは「そのブロックに変更があるか」を示すことを優先する。
- `.md` 以外の差分エントリは描画対象外とする。
- 複数の `.md` 差分エントリがある場合は、各ファイルを独立して表示する。
- 新規 `.md` は「旧ファイルなし / 新ファイルあり」として表示する。
- 削除 `.md` は「旧ファイルあり / 新ファイルなし」として表示する。

## 具体例

### テーブル差分の扱い
- NG: 変更セルだけ色付けする。
- OK: テーブル全体を 1 つの変更ブロックとしてハイライトする。

### コードブロック差分の扱い
- NG: コードブロック内の変更行だけ色付けする。
- OK: コードブロック全体を 1 つの変更ブロックとしてハイライトする。

## 受け入れ条件
- 通常テキストの追加/削除/変更が差分ビューとして判別できる。
- テーブル内に差分がある場合、テーブル全体がハイライトされる。
- コードブロック内に差分がある場合、コードブロック全体がハイライトされる。
- 出力 HTML だけで差分の意図を把握できる。
- 差分内に `.md` 以外のファイルが含まれていても、`.md` のみが描画される。
- 差分内に複数の `.md` ファイルが含まれていても、すべて描画される。
- 新規追加された `.md` ファイルの差分が正しく描画される。
- 削除された `.md` ファイルの差分が正しく描画される。

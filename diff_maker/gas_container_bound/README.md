# Container-Bound GAS

このフォルダは、1 つのスプレッドシート内で完結するコンテナバインド GAS 実装です。

## できること

- 親スプレッドシートにカスタムメニューを追加
- モーダルダイアログから複数の diff ファイルをアップロードし、1 ファイルごとに 1 シートを生成
- ダイアログで指定した出力名と同じ名前で、完成済みプレビュー HTML を Drive に保存
- 同一スプレッドシート内に 出力履歴、案件マスタ、確認者マスタ と複数の差分シートを作成
- 差分をファイル単位ではなく hunk 単位で 1 行ずつ展開
- 差分リストの各行で、案件と確認者をプルダウン選択
- 元 diff ファイルを同じ Drive フォルダへ保存
- 完成済み HTML はユーザが Drive から直接開いて確認できる

## ファイル

- appsscript.json: Apps Script マニフェスト
- Code.gs: メニュー、ダイアログ起動、diff 解析、シート生成、HTML 保存、出力履歴更新
- CreateDiffDialog.html: 複数 diff ファイル対応の作成ダイアログ

## 使い方

1. スプレッドシートを用意し、このフォルダのファイルをコンテナバインドの Apps Script プロジェクトへ配置する。
2. スクリプトを保存し、必要なら Web アプリとしてデプロイする。
3. スプレッドシートを再読込する。
4. 差分管理 > 差分リスト作成 を実行し、1 件以上の diff ファイルを選択する。
5. ファイルごとに出力名を入力する。この名前がシート名と HTML ファイル名に使われる。
6. 実行すると、diff ファイルごとに同名のシートと HTML が生成される。
7. 生成された各シートで、各差分行の 案件 と 確認者 をプルダウンから選択する。
8. 完成 HTML は Drive 上の生成ファイルをユーザが直接開いて確認する。
9. 必要に応じて 案件マスタ と 確認者マスタ を編集して候補を更新する。

## シート構成

- 出力履歴: 生成したシート名と HTML URL、diff URL の一覧。
- 案件マスタ: 案件の候補一覧。
- 確認者マスタ: 確認者の候補一覧。
- 各差分シート: diff ファイルごとに 1 枚。hunk 単位の一覧と案件・確認者入力欄を持つ。

## 注意点

- 出力名はシート名として使える文字に自動補正されます。
- 同じ出力名で再生成した場合、同名シートは上書きされ、同名の HTML / diff ファイルは既存ファイルをゴミ箱へ移したうえで再作成します。
